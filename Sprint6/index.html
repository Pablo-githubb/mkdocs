    Sprint 6 (Projecte 2 Sprint2)

    1. Gestió de processos

    1. Obre el Gestor del servidor.

    2. Ves a Tools > Task Manager.

    1. A la pestanya Details:

      • Veus el PID.

      • Fes clic dret → End Task per matar-lo.

      • Alternativament: Ctrl + Shift + Esc.

    1.1.1 Comandes

    Obre el Gestor de tasques (Task Manager) Ctrl + Shift + Esc

    O a través de PowerShell:

    Get-Process

    Visualització jeràrquica com pstree: No hi ha comanda directa, però pots veure pares/fills així:

    Get-Process | Sort-Object -Property Id | Format-Table -Property Id, Parent, ProcessName

    Per matar els processos utilitzem la comanda:

    Stop-Process -Id <PID> -Force

    2. Gestió d'usuaris i grups i permisos

    2.1 Crear nous usuaris

    En client local:

    net user alumne1 ablso330 /add
net user alumne2 ablso330 /add

    2.1.2 Versió gràfica

    1. Obre Usuarios y equipos de Active Directory

    1. A l’arbre de l'esquerra selecciona Users

    2. Clic dret → Nuevo usuario

      • Escriu el nom (alumne1), contrasenya, i marca o desmarca "El usuario debe cambiar la constraseña en el siguiente inicio de sessión"

      • Clica OK.

    2.1.3 Borrar usuaris

    A Usuarios y equipos de Active Directory, selecciona Users o Groups

    Clic dret sobre l’objecte → Eliminar

    Info

    Amb la comanda net user provaborrar /delete també podem esborrar un usuari en específic.

    2.3 Crear grups

    2.3.1 Interfície gràfica

    A Usuarios y equipos de Active Directory, selecciona Groups

    Clic dret → Nuevo Grupo

    Escriu el nom (ex: alumnesASIX)

    Clica Add → escriu el nom de l’usuari → OK

    2.3.2 Comandes

    net localgroup alumnes /add
net localgroup profes /add

    Afegir usuaris al grup:

    net localgroup alumnes alumne1 /add

    Esborrar grup:

    net localgroup profes /delete

    2.4 Modificació grups o usuaris

    • Renombrar: clic dret sobre l’usuari → Cambiar nombre

    • Canviar contrasenya: clic dret → Restablecer constraseña

    • Afegir a grups: Doble clic → Miembro de → Añadir

    2.4.1 Comandes

    Canviar nom d'usuari:

    Rename-LocalUser -Name "alumne2" -NewName "alumne4"

    Canviar contrasenya:

    net user alumne1 NovaPass123

    Note

    Assignar grup principal: no aplica com a Linux, però pots fer-ho amb permisos ACL.

    2.5 Bloqueig

    1. Torna a Users

    2. Doble clic a l’usuari → desmarca o marca "Account is disabled"

    3. Marca per bloquejar

    4. Desmarca per desbloquejar

    2.5.1 Comandes

    Per bloquejar utilitzem la comanda: 

    net user alumne3 /active:no

    Al contrari, per desbloquejar fem la comanda:

    net user alumne3 /active:yes

    2.6 Editar valors de nous usuaris

    2.6.1 Editar ruta per defecte de terminal

    En Windows no és aplicable, però pots posar scripts a l’inici de sessió amb:

    • GPO: User Configuration > Windows Settings > Scripts (Logon/Logoff)

    • Script: Set-Location "C:\Var"

    2.6.2 Afegir usuaris al grup d'administradors (sudoers)

    • Obre Usuarios y equipos de Active Directory

    • A Groups → doble clic a Admins. del dominio

    • Clica Añadir → afegeix usuari (ex: alumne1)

    2.6.3 Comandes

    net localgroup Administrators alumne1 /add

    Enllaçar

    Windows utilitza enllaços simbòlics:

    New-Item -ItemType SymbolicLink -Path "C:\Users\Default\temporalcd" -Target "C:\Temp"

    2.7 Permisos "normals" UOG

    Llenguatge octal per calcular permissos

    (-) significa que és un fitxer

    (d) significa que és un directori/carpeta

    (r) read

    (w) write

    (x) execute

    Llistar permisos:

    icacls C:\Users\alumne1

    Canviar propietari:

    takeown /F "C:\Ruta\Fitxer"

    Assignar permisos:

    icacls "C:\Ruta" /grant alumne1:(R,W)

    2.8 UMASK

    No existeix literalment, però pots aplicar permisos per defecte a carpetes base amb icacls i perfils d’usuari predeterminats (C:\Users\Default) amb la comanda següent.

    icacls "C:\Users\Default" /grant "Users:(OI)(CI)(RX)" /T

    C:\Users\Default: carpeta base de nous perfils d’usuari.

    /grant: afegeix permisos a un usuari o grup.

    "Users:(OI)(CI)(RX)": dona permisos de Lectura i Execució recursivament:

    OI = Object Inherit (fitxers dins)

    CI = Container Inherit (carpetes dins)

    RX = Read & Execute

    /T: aplica els canvis recursivament a totes les subcarpetes i fitxers.

    2.9 Permisos especials

    • Crea una carpeta (ex: C:\exercicis)

    • Botó dret → Properties → pestanya Security

    • Clica Edit

    • Clica Add → afegeix usuari o grup

    Assigna permisos: Read / Write / Modify / Full Control

    2.9.1 Comandes

    Windows no té chmod, però sí:

    icacls ex7.txt /grant alumne1:(R,W)
icacls ex8.txt /grant grup:(W)

    2.10 ACL (Access Control List)

    Afegir permisos:

    icacls "C:\exercicis" /grant vesper:(F)

    Treure permisos:

    icacls "C:\exercicis" /remove perrete

    Veure ACLs:

    icacls "C:\exercicis"

    3. Sistemes de fitxers i particions

    3.1 Teoria:

    Estructura de la informació

    • Mida sector

    • Mida block

    • Fragmentació interna: Espai desaprofitat dels blocks o clusters //solució: fer els bloc més petits

    • Fragmentación externa: Arixus guardats en molt arxius a la vegada, aixo fa que la velocitat de lectura i escriptura es reduixque.

    3.2 Tipus de formateig de particions

    Obre Disk Management (diskmgmt.msc)

    Clic dret a l'espai no assignat → New Simple Volume

    Escull la mida → Assigna una lletra → Format amb NTFS

    Marca "Perform a quick format" → Finish

    Finalment, per assignar o canviar la lletra d'unitat:

    1. Clic dret sobre la partició → Change Drive Letter and Paths

    2. Afegeix o canvia la lletra

    3.2.1 Comandes

    En primer lloc, utilitzem:

    • diskmgmt.msc (Gestor de discos)

    • diskpart (en PowerShell/CMD)

      diskpart
list disk
select disk 1
create partition primary
format fs=ntfs quick

    Windows munta automàticament les particions. Pots assignar una lletra amb:

    diskpart
select volume 3
assign letter=Z

    Note

    Es fa amb Gestió de discs o amb el registre (no cal tocar /etc/fstab com en Linux).

    3.3 Sistemes de fitxers:

    • Botó dret sobre el fitxer → Properties

    • Pestanya Security → Advanced

    • Clica Change al costat de Owner

    • Escriu el nom del nou usuari → Check Names → OK

    4. Còpies de seguretat i automatització de tasques

    4.1 Conceptes bàsics

    S'ha d'establir una bona política de còpia de seguretat

    Completa: És una còpia de seguretat la qual guarda totes les dades d'una vegada de tot el disc. És més fàcil de recuperar les dades, ja que per a restaurar les dades es necessita l'última completa. Finalment, la còpia més lenta, perquè ha de copiar tota la informació del disc dur.

    Diferencial: Solament guarda les diferències que hi ha des dee l'última còpia completa. Aquesta és més ràpida de fer a comparació de la completa perquè ha de guardar molts menys arxius i dades (tot depenent de tots els canvis que has pogut fer en un dia, però sempre serà menys que la completa). Per recuperar les dades necessites l'última còpia completa i diferencial abans de l'error al sistema.

    Incremental: És la còpia que guarda la informació de la seva anterior còpia. És molt més ràpida perquè guarda menys dades que les altres i per recuperar les dades es necessita tot el recorregut de còpies des de l'última completa fins a les anteriors a l'última incremental que has fet abans de l'error del sistema

    Warning

    Completa: utilitza robocopy o Historial de fitxers

    Diferencial: robocopy amb marca de data

    Incremental: amb programari com SyncBack, Veeam, o scripts avançats

    Còpia manual (amb Explorador)

    1. Obre File Explorer

    2. Clica dret sobre Documents → Send to > Compressed (zipped) folder

    3. Canvia el nom del fitxer amb data

    Punts de restauració

    1. Obrim l'eina de restauració: Prem les tecles Win + S, escriu "Crear un punt de restauració" i obre l'aplicació del panell de control.

    1. Configurem i creem el punt de restauració: A la finestra que apareix, selecciona la unitat del sistema (normalment C:), fes clic a "Configurar" per activar la protecció si no ho està, i després fes clic a "Crear" per guardar un punt de restauració. Això et permetrà desfer canvis futurs que puguin afectar el sistema.

    4.6 Automatització

    4.6.1 Teoría

    L’automatització a Windows consisteix en programar tasques perquè s’executin de manera automàtica, sense intervenció manual. S’utilitza per fer còpies de seguretat, manteniment, informes, neteja de fitxers temporals, etc.

    Les eines principals que tenim incorporades al nostre sistema operatiu són:

    🕒 Task Scheduler (Programador de tasques) Eina gràfica per programar scripts o programes.

    Pots definir:

    • Quan (diàriament, setmanalment, a l’inici del sistema...)

    • Què (script .ps1, .bat, programa, etc.)

    • Qui l’executa (usuari o sistema)

    💻 PowerShell

    • Permet escriure scripts potents per automatitzar tasques.

    • Exemple: còpia de seguretat, gestió d’usuaris, permisos...

    📁 Script de Logon / Logoff

    • Assignats via GPO (Polítiques de Grup).

    • S’executen quan un usuari inicia o tanca sessió.

    4.6.2 Pràctica

    1. Obre Task Scheduler (taskschd.msc)

    2. Afegeix una nova tasca:

      • General: Nom = CopiaSeguretatDiaria

      • Trigger: Daily, hora = 08:00

      • Action:

        • Programa: powershell.exe

        • Argument: -File "C:\Ruta\copia.ps1"

    Comandes

    Creem el script .ps1:

    $timestamp = Get-Date -Format "yyyyMMdd_HHmmss"
$dest = "C:\Backup\copiaseguretat_$timestamp.zip"
Compress-Archive -Path "C:\Users\pmc\Documents" -DestinationPath $dest

    D'aquesta manera ja tenim un arxiu auxiliar per al nostre sistema operatiu windows

    Com executar-lo

    1. Desa’l com Sprint2_Windows.ps1

    2. Fes clic dret → "Executa com a administrador"

    3. Si et surt la política d’execució bloquejada, obre PowerShell com a admin i executa:

      Set-ExecutionPolicy RemoteSigned

    Programem amb el TaskScheduler:

    1. Obre taskschd.msc amb Windows + R.

    2. Nova tasca → Triggers → "Cada dia a les 20:00"

    3. Acció: Executar powershell.exe -File "C:\ruta\script.ps1"
