    4.1 phpLDAPadmin

    4.1.1 Instal·lació

    1. Instal·la'l amb:

      sudo apt install phpldapadmin

    2. Edita /etc/phpldapadmin/config.php i defineix el servidor LDAP.

    Edita el fitxer de configuració amb un editor de text:

    sudo nano /etc/phpldapadmin/config.php

    Modifica la secció de configuració del servidor. Busca la línia:

    $servers->setValue('server','host','127.0.0.1');

    i canvia 127.0.0.1 per la IP o el nom del teu servidor LDAP, en el meu cas:

    $servers->setValue('server','host','ldap://10.0.2.15');

    Configura el DN d’administrador i el base DN. Busca aquestes línies i edita-les segons la teva configuració LDAP:

    $servers->setValue('login','bind_id','cn=admin,dc=Pmc,dc=cat');
$servers->setValue('server','base',array('dc=Pmc,dc=cat'));

    Permet autenticació amb contrasenya (opcional, si necessites autenticació per login):

    $servers->setValue('login','auth_type','session');

    Guarda els canvis i surt de l'editor (CTRL+X, després Y i ENTER en nano)

    Reinicia el servidor web perquè els canvis tinguin efecte:

    sudo systemctl restart apache2

    Desprès comprovem si el servei s'ha iniciat correctament.

    sudo systemctl status apache2

    Accedeix a phpLDAPadmin a través del navegador:

    http://10.0.2.15/phpldapadmin

    5. Servidor Samba

    5.1 Teoria

    Un servidor Samba, a diferència d'un servidor NFS, necessita una autenticació a nivell d'usuari. Samba permet la compartició de recursos, a més de carpetes, també impressores.

    Samba pot treballar conjuntament amb LDAP per utilitzar els usuaris per a la validació

    Samba permet clients de Windows i Linux.

    5.2 Instal·lació Samba server

    En primer lloc, utilitzem la comanda:

    apt install samba

    5.2.1 Configuració de Samba

    Al acabar d'instal·lar el servei de Samba ens dirigim a la carpeta /etc/samba/smb.conf al final de tot de l'arxiu i afegim aquests paràmetres.

    [perfils]
    path= /perfils
    guest ok = yes
    directory mask = 0755
    create mask = 0644
    browseable = yes
    read list = @dam
    write list = raul
    invalid users = enric

    Ara, per aplicar els canvis de la configuració hem d'utilitzar la comanda systemctl restart smbd nmbd i systemctl status smbd nmbd per reiniciar i veure si s'han iniciat correctament els serveis.

    systemctl restart smbd nmbd
systemctl status smbd nmbd

    5.3 Creació dels usuaris per a Samba

    A continuació, creem els usuaris que hem definit als paràmetres de configuració.

    useradd -M -s /sbin/nologin raul
useradd -M -s /sbin/nologin enric
useradd -M -s /sbin/nologin walid

    Amb tail -3 /etc/passwd podem veure els nous 3 usuaris creats recentment.

    tail -3 /etc/passwd

    Seguint amb la configuració, creem un grup anomenat "dam" amb la comanda addgroup dam. Tot seguit, afegim als usuaris creats al grup dam amb la comanda adduser usuari dam.

    adduser raul dam
adduser enric dam
adduser walid dam

    Per comprovar els canvis, utilitzem al comanda tail -1 /etc/group per veure si els usuaris s'han afegit correctament.

    5.3.1 Afegir usuaris a Samba

    Finalment, per afegir els usuaris creats a la màquina local per a Samba hem d'utilitzar la comanda smbpasswd -a usuari. Haurèm de canviar la contrasenya.

    smbpasswd -a raul
smbpasswd -a enric
smbpasswd -a walid

    5.4 Comprovació amb el client

    Per vincular el client amb el servidor Samba s'utilitza apt install smbclient.

    Obrim l'explorador d'arxius i ens dirigim a l'apartat de Otras ubicaciones. Dintre d'aqui, buscarem el nostres servidor amb la carpeta /perfils amb l'adreça smb://ipservidor/perfils (en el meu cas és smb://10.0.2.15/perfils).

    smb://10.0.2.15/perfils

    Warning

    En iniciar sessió amb un usuari recorda no modificar cap apartat que no sigui Nombre usuario o Contraseña i selecciona sempre Olvidar contraseña immediatamente per no tindre futurs problemes.

    Com anteriorment he ficat als paràmetres que pot haver-hi un usuari anònim provem a accedir amb ell.

    Provem a crear una nova carpeta i ens dona accès denegat:

    Info

    Si accedim amb un altre usuari que no tingui restriccions així podrem crear arxius i modificar-los al nostre plaer. D'altra banda, altres usuaris que tinguin restringits l'accés a la carpeta podran intentar iniciar sessió. Però no podran accedir a la carpeta compartida.

    Com podem veure, al utilitzar ls -l podem veure que el creador de la carpeta compartida de Samba és l'usuari raul:

    ls -l

    6. Perfils mòbils

    6.1 Configuració en el servidor LDAP:

    Note

    Abans fer qualsevol instal·lació, utilitzem la comanda

    sudo apt update

    Comanda inicial per instal·lar el servei de kernel: 

    apt install nfs-kernel-server

    Seguidament:

    cd /
mkdir perfils
chmod 777


    chown nobody:nogroup

    ls -l | grep perfils

    Obrim el fixer /etc/exports:

    nano /etc/exports

    I afegim aquesta línia dintre:

    /perfils *(rw,sync,no_root_squash,no_subtree_check)

    Molt important, per reiniciar el servei del kernel, utilitzem la comanda:

    systemctl restart nfs-kernel-server
systemctl status nfs-kernel-server

    Dintre de l'arxiu grup.ldif afegim les següents línies (en el meu cas):

    6.2 Configuració del client LDAP

    Note

    Abans fer qualsevol instal·lació, utilitzem la comanda

    sudo apt update

    sudo apt install nfs-common rpcbind

    Seguidament:

    cd /
mkdir perfils
chmod 777
ls -l | grep perfils

    Entrem a configurar l'arxiu /etc/fstab amb la comanda:

    nano /etc/fstab

    A més, dintre de l'arxiu afegim la següent llínea: 

    10.0.2.15:/perfils /perfils nfs auto,noatime,nolock,bg,nfsvers=3,intr,tcp,actimeo=1800 0 0

    Per acabar, reiniciem la màquina per comprovar si s'han aplicat els canvis.
